THIS FOLDER CONTAINS:
  frames/      →  480 WebP image files (f0000.webp … f0479.webp)
                  These are the frames of the hero animation, served
                  individually so the browser can cache and stream them.
  frames.js    →  Auto-generated list of frame paths. Loaded by the site
                  to know which images to fetch. Do NOT edit by hand.
  convert.sh   →  Bash script to re-convert PNG frames → WebP using FFmpeg.
                  Run this when you have a new animation to swap in.

  HOW TO REPLACE THE ANIMATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  When you are ready to swap in a new animation, ask Claude Code
  (or whoever builds your site) to:

    "Replace the hero animation in the ENN website.
     Here is my new source: [attach file or link your frames folder]
     The animation lives in  ANIMATION/frames/"

  They will:
    1. Export the source as a PNG sequence (ideally 300–500 frames @ 4K)
    2. Run  ANIMATION/convert.sh  to convert → 1920×1080 WebP quality 92
       (or run the Python converter if FFmpeg is not installed)
    3. The script regenerates  ANIMATION/frames.js  automatically
    4. You delete the old frames/ folder, copy in the new one, commit & push

  WHAT NOT TO TOUCH
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  • Do NOT edit frames.js by hand — it is machine-generated.
  • Do NOT rename the webp files — names must stay f0000.webp, f0001.webp…
  • Do NOT delete .nojekyll from the repo root — it is required for
    GitHub Pages to serve binary files correctly.
